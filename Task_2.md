# Task 2 — Advanced Airtable Automation

In Task 2, you will build an **automation** and an **interface** for the HR onboarding process.

### 1. Build Automation Logic:
- **Trigger**: The automation should trigger when a new hire is categorized as "High Value" in Task 1.
- **Slack/Email Notification**: Send a dynamic Slack/Email notification to HR when a new hire is marked as "High Value."
- **Tracking Table**: Log the "High Value" hire into a **Tracking Table**.
- **Conditional Branching**: Ensure that if the Email is missing or invalid, appropriate steps are taken (e.g., send a notification to HR or create a separate task).

### 2. Build an Interface:
Create an **interface** in Airtable with the following:
- **Summary Tiles**: Display counts of High, Medium, and Low Value new hires.
- **Filtered Grid**: A grid of new hires marked as "High Value."
- **Detail View**: A detailed view for each new hire with their Full Name, Email, Equipment Amount, etc.

### Deliverables:
1. Screenshots of your interface design.
2. A description of the **automation steps**.
3. **Formula logic** for any conditions used in the automation.
4. Any assumptions or explanations you used in your solution.

Submit all deliverables in the `submissions/` folder.
